# 按键动作

按键动作用于在径向菜单里触发已有按键绑定（原版或模组）。

## 字段

- **标题**：编辑器和径向菜单里显示的名称。
- **备注**：可选说明。
- **映射名称**：按键映射 ID 或标签。
- **触发方式**：`AUTO`、`INPUT`、`TICK`。
- **切换**：开启后不再是单次点击，而是切换按下/松开状态。
- **图标**：点击图标框打开图标选择器。

## 推荐设置流程

1. 点击 `Pick from Keybinds`。
2. 从列表选择目标按键（比手填更安全）。
3. 无特殊需求时保持 `AUTO`。
4. 保存。

## 触发方式说明

| 模式 | 行为 | 适用场景 |
|---|---|---|
| `AUTO` | 自动选择最合适路径 | 默认首选 |
| `INPUT` | 通过输入管线注入按键事件 | `AUTO` 不生效时尝试 |
| `TICK` | 通过按键状态逐 tick 更新 | 难注入按键的兜底方案 |

## 切换（Toggle）

- `OFF`：每次触发执行一次点击。
- `ON`：每次触发在按下 / 松开之间切换。

适合冲刺、潜行等“保持状态型”按键（具体取决于目标按键逻辑）。

## 常见问题

- **没有反应：** 检查映射 ID，或重新从按键列表选择。
- **触发了错误按键：** 手填名字可能匹配错，建议用选择器。
- **单机可用但服务器无效：** 目标行为可能受服务器限制。

???+ info "深入说明：按键投递解析"
    内部会按以下优先级解析映射：

    1. 精确匹配翻译键
    2. 精确匹配本地化标签
    3. 包含匹配兜底

    对需要修饰键（Ctrl/Shift/Alt）的按键会使用额外注入逻辑。`AUTO` 会根据按键绑定类型、scancode、修饰键需求自动切策略。
