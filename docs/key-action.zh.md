# 关键动作

使用按键操作从径向触发现有的按键绑定（普通或修改）。

## 字段

- **标题**：您在编辑器/径向标签中看到的内容。
- **注意**：可选的帮助文本。
- **映射名称**：键映射 ID 或标签。
- **交付**：`AUTO`、`INPUT` 或 `TICK`。
- **切换**：切换按键状态而不是点击。
- **图标**：单击图标框打开图标选择器。

## 推荐设置

1. 单击`Pick from Keybinds`。
2. 从选择器中选择绑定（比手动输入更安全）。
3. 除非有特殊原因，否则请继续通过`AUTO` 进行交付。
4. 保存。

## 交付方式

|模式|它有什么作用 |何时使用 |
|---|---|---|
| `AUTO` |自动选择最佳路径 |几乎每个人的默认设置|
| @@代码1@@ |通过输入管道注入按键按下/释放 |如果 `AUTO` | 中的绑定没有响应
| `TICK` |通过按键状态更新设置按键向下/向上 |对于难以插入的密钥的有用后备方案 |

## 切换

- `OFF`：每次径向使用时轻按一下。
- `ON`：每次使用时向下/向上翻转按键。

对于冲刺/潜行风格切换等操作很有用，具体取决于目标按键绑定的行为方式。

## 常见问题

- **没有任何反应：** 检查映射 ID 或再次从按键绑定列表中选择。
- **错误的按键绑定触发器：**您输入的名称与另一个映射标签匹配；使用选择器。
- **适用于单人游戏，但不适用于服务器：**目标操作可能受服务器限制。

???+ 信息“深入探讨：密钥传送的工作原理”
    在内部，EZ Actions 通过以下方式解析映射名称：

1.精确翻译键匹配
    2. 精确的本地化标签匹配
    3.后备包含匹配

对于需要修饰符的键（Ctrl/Shift/Alt），它使用额外的注入逻辑。 `AUTO` 可以根据按键是否已绑定、是否有扫描码或是否需要修饰符来切换策略。
