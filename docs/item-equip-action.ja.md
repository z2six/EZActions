# アイテム装備アクション

アイテム装備を使用すると、ギアのスナップショットを保存し、インベントリから一致するアイテムを再装備できます。

## ターゲット スロット

次のいずれかを割り当てることができます。

- メインハンド
- 率直な感想
- ヘルメット
- チェストプレート
- レギンス
- ブーツ

エディターでスロットが空のままの場合、EZ アクションは実行中にそのスロットをスキップします。

## 作成方法

1. 「`Add Item Equip`」をクリックします。
2. タイトル/メモ/アイコンを設定します。
3. アイテムをソース グリッドからターゲット スロットにドラッグします。
4. 保存します。

### ソースグリッドに含まれるもの

- 現在の手持ちアイテム
- 装備された鎧
- 主な在庫
- ホットバー

## 実行中

ラジアルからアクションをトリガーすると:

1. EZ アクションは、設定されている各ターゲット スロットをチェックします。
2. ターゲットがすでに記録されている項目と一致する場合、スキップします。
3. そうでない場合は、最も一致するソース スタックが検索されます。
4. アイテムをターゲットスロットに交換します。

スロットごとに処理し、部分的な成功を許可します。

## 重要な一致ルール

- マッチングはフルスタック スナップショット署名 (NBT およびメタデータ) に基づいており、カウントは無視されます。
- 一致するスタックが複数存在する場合は、カウントが最も大きいスタックが選択されます。

## メインハンドのルール

`Mainhand` は、トリガー時に現在選択されているホットバー スロットを意味します。

## プレッシャーの下での行動

- 1 つのアイテム装備アクションの実行中に 2 つ目のアイテム装備アクションをトリガーすると、古いアクションはキャンセルされ、置き換えられます。
- 実行がバックグラウンドティックで実行されている間、動きとゲームプレイの入力はアクティブなままになります。

## エディターのクイック コントロール

- ソースからターゲット スロットまで LMB ドラッグ: 項目を割り当てます。
- ターゲットスロットの RMB: 割り当てをクリアします。
- `Refresh Items`: 現在のプレーヤーのインベントリ状態からソース リストを再構築します。

???+ 情報「詳細: スロット実行順序」
    現在の実行順序は次のとおりです。

1. ヘルメット
    2.胸当て
    3. レギンス
    4. ブーツ
    5. 率直な意見
    6. メインハンド

空のターゲットと一致しないソースはスキップされ、ハード障害として扱われません。
