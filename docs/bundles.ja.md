# バンドル

バンドルは、放射状ツリー内のフォルダー (カテゴリ) です。

これらを使用して、戦闘、建築、ユーティリティ、ロールプレイなどのコンテキスト別にアクションをグループ化します。

## バンドルフィールド

- **タイトル** (内部IDとしても使用)
- **注** (オプション)
- **アイコン**
- **メインラジアルから隠す**
- **キーバインドを有効にする**

## バンドルのキーバインド

`Enable keybind` がオンの場合、EZ Actions はそのバンドルの専用キーバインドを登録します。

!!!警告「再起動が必要です」
    バンドル キーバインドの登録は、次回のクライアントの再起動時に適用されます。

EZ アクションは、再起動が必要な場合にクライアント メッセージを表示します (API で作成されたバンドルを含む)。

## メインラジアルから隠す

有効にした場合:

- ルートラジアルページではバンドルが非表示になります。
- バンドルはメニュー モデルにまだ存在します。
- バンドルは API またはバンドル キーバインド経由で開くことができます。

ルートを乱雑にしたくない「高度なページ」に適しています。

## ネストされたバンドル

バンドルには以下を含めることができます。

- 主要なアクション
- コマンドアクション
- アイテム装備アクション
- より多くのバンドル

## ベストプラクティス

- root を小さく、優先度を高く保ちます。
- 低頻度のアクションをより深いバンドルに配置します。
- バンドルに明確なアイコンと短い名前を付けます。

## ロックされたバンドル

バンドルには `locked` のマークを付けることができます (通常は API または JSON 経由)。

- ゲーム内の削除パスでは削除されません。
- 手動で JSON を編集すると、削除される可能性があります。

???+ 情報「詳細: アイデンティティと独自性」
    バンドルのタイトルは、エディターのワークフローでバンドル ID として使用されます。

バンドル名が重複するとあいまいさが生じる可能性があるため、エディターは重複したバンドル タイトル/ID の保存をブロックします。
