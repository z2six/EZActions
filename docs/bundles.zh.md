# 捆绑包

捆绑包是径向树内的文件夹（类别）。

使用它们按上下文对操作进行分组：战斗、建筑、实用、角色扮演等。

## 捆绑字段

- **标题**（也用作内部 ID）
- **注意**（可选）
- **图标**
- **隐藏主径向**
- **启用按键绑定**

## 捆绑按键绑定

如果 `Enable keybind` 已打开，EZ Actions 会为该捆绑包注册专用按键绑定。

!!!警告“需要重新启动”
    捆绑键绑定注册将在下次客户端重新启动时应用。

当需要重新启动时，EZ Actions 会显示一条客户端消息（包括 API 创建的捆绑包）。

## 隐藏主径向线

如果启用：

- 捆绑包隐藏在根径向页面上。
- 捆绑包仍然存在于菜单模型中。
- 捆绑包仍然可以通过 API 或捆绑包键绑定打开。

适合“高级页面”，您不希望根目录混乱。

## 嵌套捆绑包

捆绑包可以包含：

- 关键行动
- 命令动作
- 物品装备动作
- 更多捆绑包

## 最佳实践

- 保持根小且优先级高。
- 将低频动作放入更深的捆绑中。
- 为捆绑包提供清晰的图标和简称。

## 锁定捆绑包

捆绑包可以标记为 `locked`（通常通过 API 或 JSON）。

- 游戏内删除路径不会将其删除。
- 手动 JSON 编辑仍然可以将其删除。

???+ 信息“深入探讨：身份和独特性”
    捆绑包标题在编辑器工作流程中用作捆绑包 ID。

重复的捆绑包名称可能会导致歧义，因此编辑器会阻止重复的捆绑包标题/ID 保存。
