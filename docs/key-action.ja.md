# キーアクション

キー アクションを使用して、ラジアルから既存のキーバインド (バニラまたは改造) を起動します。

## フィールド

- **タイトル**: エディター/ラジアル ラベルに表示されるもの。
- **注意**: オプションのヘルパー テキスト。
- **マッピング名**: キー マッピング ID またはラベル。
- **配信**: `AUTO`、`INPUT`、または `TICK`。
- **切り替え**: タップの代わりにキーの状態を切り替えます。
- **アイコン**: アイコン ボックスをクリックしてアイコン ピッカーを開きます。

## 推奨されるセットアップ

1. 「`Pick from Keybinds`」をクリックします。
2. ピッカーからバインディングを選択します (手で入力するより安全です)。
3. 特別な理由がない限り、`AUTO` で配信を続けます。
4. 保存します。

## 配信モード

|モード |何をするのか |いつ使用するか |
|---|---|---|
| `AUTO` |最適なパスを自動的に選択します |ほぼ全員のデフォルト |
| `INPUT` |入力パイプラインを介してキーの押下/解放を挿入します。バインディングが `AUTO` で応答しない場合 |
| `TICK` |キー状態の更新を介してキーをダウン/アップに設定します。注入が難しいキーの便利なフォールバック |

## トグル

- `OFF`: ラジアル使用ごとに 1 回タップします。
- `ON`: 使用するたびにキーを上下に切り替えます。

ターゲットのキーバインドの動作に応じて、スプリント/スニーク スタイルの切り替えなどのアクションに役立ちます。

## よくある問題

- **何も起こりません:** マッピング ID を確認するか、キーバインド リストから再度選択します。
- **間違ったキーバインド トリガー:** 入力した名前が別のマッピング ラベルと一致しました。ピッカーを使用します。
- **シングルプレイヤーでは動作しますが、サーバーでは動作しません:** ターゲット アクションはサーバーによって制限されている可能性があります。

???+ 情報「詳細: キー配信の仕組み」
    内部的には、EZ アクションは次の方法でマッピング名を解決します。

1. 翻訳キーの正確な一致
    2. ローカライズされたラベルの正確な一致
    3. フォールバックには一致が含まれます

修飾子が必要なキー (Ctrl/Shift/Alt) の場合、追加の挿入ロジックが使用されます。 `AUTO` は、キーがバインドされているか、スキャンコードがあるか、または修飾子が必要かどうかに応じて戦略を切り替えることができます。
