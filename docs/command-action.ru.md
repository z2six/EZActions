# Команда Действие

Используйте командное действие для отправки команд по радиальной линии.

## Поля

- **Название**
- **Примечание**
- **Команда** (многострочная)
- **Задержка нескольких команд (тики)**
- **Команды цикла (одна на использование)**
- **Значок**

## Правила командного поля

- Одна строка = одна команда.
- Начальный `/` не является обязательным (EZ Actions удаляет его перед отправкой).
- Пустые строки игнорируются.

## Поведение задержки

`Multi-command delay (ticks)` управляет расстоянием между командными строками, когда они не циклически повторяются.

- `0`: отправлять строки немедленно.
- `>0`: поставить в очередь построчно с такой задержкой.

## Команды цикла

Если этот параметр включен, каждое радиальное использование отправляет ровно одну строку и переходит к следующей.

Пример:

```text
/time set day
/time set night
```

Use #1 -> day  
Use #2 -> night  
Use #3 -> day

## Практические примеры использования

- Быстрые служебные команды (`/home`, `/spawn`, `/warp mine`)
- Переключение ролевой игры (`/hat`, `/nick`)
- Рабочие процессы администратора разделены по строкам.

## Примечания

— Это диспетчеризация на стороне клиента: разрешения сервера по-прежнему применяются.
- Если начинается новая последовательность команд, предыдущая последовательность в очереди заменяется.

???+ информация «Глубокое погружение: модель секвенирования»
    — Нециклические многострочные команды используют клиентский секвенсор тиков.
    - В циклическом режиме в экземпляре действия сохраняется внутренний курсор.
    - Циклический и немедленный режимы отменяют последовательность действий в полете перед отправкой.
