# 명령 동작

방사형에서 명령을 보내려면 명령 작업을 사용하십시오.

## 필드

- **제목**
- **참고**
- **명령**(여러 줄)
- **다중 명령 지연(틱)**
- **주기 명령(사용당 하나)**
- **아이콘**

## 명령 상자 규칙

- 한 줄 = 하나의 명령.
- `/` 앞부분은 선택 사항입니다(EZ Actions는 보내기 전에 이를 제거합니다).
- 빈 줄은 무시됩니다.

## 지연 동작

`Multi-command delay (ticks)`은 순환하지 않을 때 명령줄 사이의 간격을 제어합니다.

- `0`: 즉시 라인을 보냅니다.
- `>0`: 해당 지연을 포함하여 한 줄씩 대기열에 추가됩니다.

## 사이클 명령

활성화된 경우 각 방사형 사용은 정확히 한 줄을 보내고 다음 줄로 회전합니다.

예:

```text
/time set day
/time set night
```

Use #1 -> day  
Use #2 -> night  
Use #3 -> day

## 실제 사용 사례

- 빠른 유틸리티 명령(`/home`, `/spawn`, `/warp mine`)
- 역할극 토글(`/hat`, `/nick`)
- 여러 라인으로 분할된 관리 작업 흐름

## 메모

- 이것은 클라이언트측 디스패치입니다. 서버 권한은 여전히 ​​적용됩니다.
- 새 명령 시퀀스가 ​​시작되면 이전에 대기 중인 시퀀스가 ​​교체됩니다.

    - 사이클링 및 즉시 모드는 파견 전에 기내 시퀀스를 취소합니다.
