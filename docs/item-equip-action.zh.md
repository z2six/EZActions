# 物品装备行动

物品装备可让您保存装备快照并重新装备库存中的匹配物品。

## 目标槽位

您可以分配其中任何一个：

- 主手
- 副手
- 头盔
- 胸甲
- 打底裤
- 靴子

如果编辑器中的某个槽留为空，EZ Actions 在执行期间会跳过它。

## 如何创建一个

1. 单击`Add Item Equip`。
2. 设置标题/注释/图标。
3. 将项目从源网格拖动到目标插槽中。
4. 保存。

### 源网格包括

- 当前副手物品
- 装备盔甲
- 主要库存
- 快捷栏

## 执行期间

当您从径向触发操作时：

1. EZ Actions 检查每个配置的目标插槽。
2. 如果目标已经与记录的项目匹配，则跳过。
3. 如果没有，则找到最佳匹配的源堆栈。
4. 将物品交换到目标插槽中。

它逐个时隙进行处理并允许部分成功。

## 重要的匹配规则

- 匹配基于全堆栈快照签名（NBT 和元数据），忽略计数。
- 如果存在多个匹配的堆栈，它将选择计数最高的一个。

## 主手规则

`Mainhand` 表示您当前在触发时选择的热键栏插槽。

## 压力下的行为

- 如果您在一个物品装备操作正在运行时触发第二个物品装备操作，则旧的物品装备操作将被取消并替换。
- 当执行在后台运行时，移动和游戏输入保持活动状态。

## 编辑器中的快速控制

- LMB 从源插槽拖动到目标插槽：分配项目。
- 目标位置上的人民币：明确分配。
- `Refresh Items`：从当前玩家库存状态重建源列表。

???+ 信息“深入探讨：插槽执行顺序”

1. 头盔
    2. 胸甲

空目标和丢失的源匹配将被跳过，不被视为硬故障。
