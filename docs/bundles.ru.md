# Пакеты

Пакеты — это папки (категории) внутри радиального дерева.

Используйте их для группировки действий по контексту: бой, строительство, полезность, ролевая игра и т. д.

## Поля пакета

- **Название** (также используется как внутренний идентификатор)
- **Примечание** (необязательно)
- **Значок**
- **Скрыть от основного радиала**
- **Включить привязку клавиш**

## Пакет сочетаний клавиш

Если `Enable keybind` включен, EZ Actions регистрирует специальную привязку клавиш для этого пакета.

!!! предупреждение «Требуется перезагрузка»
    Регистрация привязки клавиш пакета применяется при следующем перезапуске клиента.

EZ Actions отображает сообщение клиента при необходимости перезапуска (включая пакеты, созданные API).

## Скрыть от главного радиала

Если включено:

- Пакет скрыт на корневой радиальной странице.
- Пакет все еще существует в модели меню.
- Пакет по-прежнему можно открыть через API или привязку клавиш пакета.

Хорошо для «расширенных страниц», вы не хотите загромождать корень.

## Вложенные пакеты

Пакеты могут содержать:

- ключевые действия
- командные действия
- действия по экипировке предметов
- больше пакетов

## Лучшие практики

- Держите root небольшим и высокоприоритетным.
- Поместите низкочастотные действия в более глубокие связки.
- Дайте пакетам понятные значки и короткие названия.

## Заблокированные пакеты

Пакет может быть помечен `locked` (обычно через API или JSON).

- Пути удаления в игре не удалят его.
- Ручное редактирование JSON все равно может удалить его.

???+ информация «Глубокое погружение: идентичность и уникальность»
    Название пакета используется в качестве идентификатора пакета в рабочих процессах редактора.

Повторяющиеся имена пакетов могут привести к двусмысленности, поэтому редактор блокирует сохранение повторяющихся названий/идентификаторов пакетов.
