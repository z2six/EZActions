# 命令动作

命令动作用于在径向菜单里发送命令。

## 字段

- **标题**
- **备注**
- **命令**（支持多行）
- **多命令延迟（ticks）**
- **循环命令（每次使用执行下一条）**
- **图标**

## 命令框规则

- 一行就是一条命令。
- 开头的 `/` 可写可不写（EZ Actions 发送前会处理）。
- 空行会被忽略。

## 延迟行为

`Multi-command delay (ticks)` 控制非循环模式下多行命令的间隔。

- `0`：立即发送所有行。
- `>0`：按延迟逐行发送。

## 循环命令

开启后，每次触发只发送一行，并轮转到下一行。

示例：

```text
/time set day
/time set night
```

第 1 次 -> day  
第 2 次 -> night  
第 3 次 -> day

## 实用场景

- 常用工具命令（`/home`、`/spawn`、`/warp mine`）
- 角色扮演切换（`/hat`、`/nick`）
- 管理命令流程拆分

## 注意事项

- 命令分发是客户端侧发起，是否可执行仍受服务器权限控制。
- 当新命令序列开始时，旧的排队序列会被替换。

???+ info "深入说明：命令时序模型"
    - 非循环多行命令使用客户端 tick 时序器。
    - 循环模式在动作实例内部保存游标。
    - 循环与即时模式在分发前都会取消当前在途序列。
