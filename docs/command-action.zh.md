# 命令动作

使用命令操作从径向发送命令。

## 字段

- **标题**
- **注意**
- **命令**（多行）
- **多命令延迟（滴答声）**
- **循环命令（每次使用一个）**
- **图标**

## 命令框规则

- 一行 = 一个命令。
- 前导`/` 是可选的（EZ Actions 在发送之前将其剥离）。
- 空行被忽略。

## 延迟行为

`Multi-command delay (ticks)` 控制不循环时命令行之间的间距。

- `0`：立即发送行。
- `>0`：按该延迟逐行排队。

## 循环命令

如果启用，每次径向使用只会发送一行并旋转到下一行。

例子：

```text
/time set day
/time set night
```

Use #1 -> day  
Use #2 -> night  
Use #3 -> day

## 实际用例

- 快速实用命令（`/home`、`/spawn`、`/warp mine`）
- 角色扮演切换（`/hat`、`/nick`）
- 管理工作流程跨线划分

## 注释

- 这是客户端调度：服务器权限仍然适用。
- 如果新的命令序列开始，则先前排队的序列将被替换。

???+ 信息“深入研究：测序模型”
