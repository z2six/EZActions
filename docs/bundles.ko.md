# 번들

번들은 방사형 트리 내부의 폴더(범주)입니다.

전투, 건물, 유틸리티, 역할극 등 상황에 따라 작업을 그룹화하는 데 사용합니다.

## 번들 필드

- **제목**(내부 ID로도 사용됨)
- **참고**(선택사항)
- **아이콘**
- **주 방사형에서 숨기기**
- **키 바인딩 활성화**

## 번들 키 바인딩

`Enable keybind`이 켜져 있으면 EZ Actions는 해당 번들에 대한 전용 키 바인딩을 등록합니다.

!!! "다시 시작해야 합니다" 경고
    번들 키 바인딩 등록은 다음 클라이언트 재시작 시 적용됩니다.

EZ Actions는 재시작이 필요할 때 클라이언트 메시지를 표시합니다(API로 생성된 번들 포함).

## 메인 방사형에서 숨기기

활성화된 경우:

- 번들은 루트 방사형 페이지에 숨겨져 있습니다.
- 메뉴 모델에는 번들이 여전히 존재합니다.
- 번들은 API나 번들 키 바인딩을 통해 계속 열 수 있습니다.

복잡한 루트를 원하지 않는 "고급 페이지"에 적합합니다.

## 중첩된 번들

번들에는 다음이 포함될 수 있습니다.

- 주요 조치
- 명령 동작
- 아이템 장착 액션
- 더 많은 묶음

## 모범 사례

- 루트를 작고 높은 우선순위로 유지하세요.
- 저주파 작업을 더 깊은 묶음에 넣습니다.
- 번들에 명확한 아이콘과 짧은 이름을 제공합니다.

## 잠긴 번들

번들은 `locked`으로 표시될 수 있습니다(일반적으로 API 또는 JSON을 통해).

- 게임 내 삭제 경로에서는 삭제되지 않습니다.
- 수동 JSON 편집으로 제거할 수 있습니다.

???+ info "심층 분석: 정체성과 고유성"
    번들 제목은 편집기 작업 흐름에서 번들 ID로 사용됩니다.

중복된 번들 이름은 모호성을 유발할 수 있으므로 편집기는 중복된 번들 제목/ID 저장을 차단합니다.
